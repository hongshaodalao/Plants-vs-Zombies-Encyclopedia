# PvZ 2 中国版扩展 - 设计文档

## 1. 项目概述

### 1.1 项目目标

在现有 PvZ 2 国际版百科基础上，扩展收录中国版独占内容，包括独占世界、植物和僵尸。

### 1.2 内容范围

- 中国版独占世界（西游记、功夫世界、天空之城等）
- 中国版独占植物（火葫芦、白萝卜等）
- 中国版独占僵尸（牛魔王僵尸等）

### 1.3 数据来源

中文 PvZ Wiki（plantsvszombies.fandom.com/zh）

---

## 2. 数据模型

### 2.1 字段扩展

在现有 PvZ 2 数据结构中添加 `source` 字段：

```javascript
{
  id: "peashooter",
  name: "豌豆射手",
  nameEn: "Peashooter",
  source: "international",  // 新增：数据来源
  // ... 其他字段保持不变
}
```

**source 值**：
- `"international"` — 国际版内容
- `"china"` — 中国版独占内容

### 2.2 中国版独占世界（预计）

| 世界 ID | 中文名 | 英文名 |
|---------|--------|--------|
| `journey_to_the_west` | 西游记 | Journey to the West |

| `sky_city` | 天空之城 | Sky City |
| `steam_ages` | 蒸汽时代 | Steam Ages |
| `renaissance_age` | 文艺复兴 | Renaissance Age |
| `heian_age` | 平安时代 | Heian Age |
| `dino_world` | 恐龙时代 | Dino World |
| `md_eras` | 全新纪元 | MD Eras |

> 注：具体世界列表以实际数据采集结果为准

### 2.3 数据文件结构

现有数据文件扩展：
- `src/data/pvz2/plants.js` — 添加中国版植物
- `src/data/pvz2/zombies.js` — 添加中国版僵尸
- `src/data/pvz2/worlds.js` — 添加中国版世界

## 3. UI 设计

### 3.1 卡片标签

在植物/僵尸卡片上显示来源标签：

- 中国版：红色背景 + 白色文字 `[中国版]`
- 国际版：不显示标签（默认）

### 3.2 筛选面板扩展

在 PvZ 2 筛选面板中添加"来源"筛选项：

```
分类: [射手] [爆炸] [防御] [辅助]
来源: [国际版] [中国版]  ← 新增
世界: [神秘埃及] [海盗港湾] ...
```

### 3.3 详情页扩展

在详情页中显示来源信息：

- 中国版独占：显示 `🇨🇳 中国版独占`
- 国际版：不显示来源标签

## 4. 分阶段实施计划

### 阶段 1：数据模型扩展（预计 0.5 天）

- 在现有 PvZ 2 数据结构中添加 `source` 字段
- 为现有国际版内容设置 `source: "international"`
- 更新筛选面板组件支持"来源"筛选

### 阶段 2：中国版世界数据（预计 1 天）

- 从中文 PvZ Wiki 采集中国版独占世界数据
- 下载世界图片
- 更新 `src/data/pvz2/worlds.js`

### 阶段 3：中国版植物数据（预计 2 天）

- 从中文 PvZ Wiki 采集中国版独占植物数据（预计 50-100 种）
- 下载植物图片
- 更新 `src/data/pvz2/plants.js`

### 阶段 4：中国版僵尸数据（预计 2 天）

- 从中文 PvZ Wiki 采集中国版独占僵尸数据（预计 30-60 种）
- 下载僵尸图片
- 更新 `src/data/pvz2/zombies.js`

### 阶段 5：UI 更新与测试（预计 1 天）

- 在卡片组件添加来源标签
- 在详情页添加来源信息
- 更新筛选面板
- 测试所有功能
- 更新 README

**总计预计：6.5 天**

---

## 5. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 中国版数据量不确定 | 时间估算偏差 | 先采集世界数据，再估算植物/僵尸数量 |
| 中文 Wiki 数据不完整 | 部分字段缺失 | 使用 null 填充，后续补充 |
| 图片版权 | 法律风险 | 仅个人学习使用 |
| 数据格式不一致 | 解析困难 | 使用 MediaWiki API 统一获取 |
