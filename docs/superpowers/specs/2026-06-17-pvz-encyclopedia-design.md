# 植物大战僵尸百科全书 - 设计文档

## 1. 项目概述

### 1.1 项目目标

构建一个植物大战僵尸（PvZ 1，2009年）的在线百科全书网站，提供植物和僵尸的详细图鉴信息，支持搜索、筛选和排序功能。

### 1.2 目标用户

- PvZ 玩家，希望快速查阅游戏数据
- 个人学习/娱乐用途

### 1.3 核心功能

- 植物图鉴：展示所有植物的属性、技能、图片
- 僵尸图鉴：展示所有僵尸的属性、弱点、图片
- 搜索：按名称实时搜索
- 筛选：按分类、属性等条件过滤
- 排序：按数值型属性排序
- 详情页：展示完整信息

## 2. 技术架构

### 2.1 技术栈

| 层级 | 技术 | 用途 |
|------|------|------|
| 构建工具 | Vite | 快速开发、热更新 |
| 前端框架 | React 18 | UI 组件化 |
| 路由 | React Router v6 | 页面导航 |
| 样式 | CSS Modules | 组件级样式隔离 |

## 3. 数据模型

### 3.1 植物数据结构

```javascript
{
  id: "peashooter",           // 唯一标识，用于路由
  name: "豌豆射手",            // 中文名称
  nameEn: "Peashooter",       // 英文名称
  sunCost: 100,               // 阳光消耗
  recharge: 7.5,              // 冷却时间（秒）
  damage: 20,                 // 伤害值
  health: 300,                // 生命值
  range: "straight",          // 攻击范围类型
  category: "shooter",        // 分类
  unlockLevel: 1,             // 解锁关卡
  description: "发射豌豆攻击僵尸",
  image: "/images/plants/peashooter.png"
}
```

**分类枚举**：
- `shooter` - 射手类
- `explosive` - 爆炸类
- `defensive` - 防御类
- `support` - 辅助类

### 3.2 僵尸数据结构

```javascript
{
  id: "zombie",               // 唯一标识
  name: "普通僵尸",            // 中文名称
  nameEn: "Zombie",           // 英文名称
  health: 200,                // 生命值
  speed: "slow",              // 移动速度
  damage: 100,                // 攻击伤害
  category: "basic",          // 分类
  firstAppearance: 1,         // 首次出现关卡
  weakness: "any",            // 弱点
  description: "最基础的僵尸",
  image: "/images/zombies/zombie.png"
}
```

**分类枚举**：
- `basic` - 基础类
- `cone` - 路障类
- `bucket` - 铁桶类
- `special` - 特殊类

**速度枚举**：
- `slow` - 慢
- `medium` - 中
- `fast` - 快

## 5. 搜索与筛选系统

### 5.1 搜索功能

- 实时搜索，输入即筛选
- 搜索范围：中文名称、英文名称、描述
- 300ms 防抖
- 匹配文字高亮

### 5.2 筛选条件

**植物筛选**：

| 筛选项 | 选项 |
|--------|------|
| 分类 | 射手类、爆炸类、防御类、辅助类 |
| 阳光消耗 | 0-50、51-100、101-200、200+ |
| 解锁关卡 | 1-10、11-20、21-30、31-50 |

**僵尸筛选**：

| 筛选项 | 选项 |
|--------|------|
| 分类 | 基础类、路障类、铁桶类、特殊类 |
| 移动速度 | 慢、中、快 |
| 首次出现 | 1-10、11-20、21-30、31-50 |

### 5.3 排序功能

**植物排序**：阳光消耗、生命值、解锁关卡（升序/降序）

**僵尸排序**：生命值、首次出现关卡（升序/降序）

### 5.4 筛选逻辑

- 多条件：AND 关系
- 同选项内多选：OR 关系
- 实时结果计数："显示 X/Y 个结果"
- 重置筛选按钮

## 6. 组件设计

### 6.1 Layout 组件

- 顶部导航栏：Logo + 链接（植物/僵尸）
- 内容区域：`<Outlet />` 渲染子路由
- 底部版权信息

### 6.2 Card 组件

**Props**：
- `data`: 植物/僵尸数据对象
- `type`: "plant" | "zombie"

**展示内容**：
- 缩略图
- 名称（中文）
- 关键属性（阳光消耗/生命值）

**交互**：
- 使用 `Link` 包裹，点击跳转详情
- 响应式网格：移动端 1 列，平板 2 列，桌面 3 列

### 6.3 SearchBar 组件

**Props**：
- `value`: 搜索关键词
- `onChange`: 变更回调

**功能**：
- 输入框 + 搜索图标
- 清除按钮
- 300ms 防抖

### 6.4 FilterPanel 组件

**Props**：
- `filters`: 当前筛选条件
- `onFilterChange`: 筛选变更回调
- `type`: "plant" | "zombie"

**功能**：
- 可折叠/展开
- Checkbox 多选
- 结果计数
- 重置按钮

### 6.5 Detail 组件

**Props**：
- `data`: 完整数据对象
- `type`: "plant" | "zombie"

**展示内容**：
- 大图
- 所有属性列表
- 描述文字
- 返回按钮
- 前后导航

## 7. 图片资源

### 7.1 图片来源

从 PvZ Wiki 等公开资源获取，仅用于个人学习用途。

### 7.2 图片规格

- 格式：PNG（带透明背景）
- 尺寸：200x200 像素（缩略图）、400x400 像素（详情图）
- 命名：与数据 `id` 一致（如 `peashooter.png`）

### 7.3 图片优化

- 使用 `loading="lazy"` 懒加载
- 考虑使用 WebP 格式（可选）

## 8. 性能优化

- 图片懒加载
- 搜索防抖（300ms）
- 路由懒加载（`React.lazy`）
- 列表虚拟化（如数据量大，使用 `react-window`）
